# 🚀 jihotv 웹매거진 리뉴얼 실행 계획 (Blueprint)

`new-plan.md` 기획서에 명시된 목표를 달성하기 위해, 프로젝트를 5개의 주요 단계로 나누어 실행합니다.

---

### **[1단계] 프로젝트 기반 구축 및 기본 레이아웃 설정**

**목표:** 새로운 Next.js 프로젝트를 초기화하고, 디자인 시스템의 핵심(색상, 타이포그래피, 간격)을 적용하여 기본적인 페이지 레이아웃을 구축합니다.

**상세 작업 내용:**
1.  **프로젝트 초기화:**
    *   기존의 정적 HTML 파일들(`index.html`, `about.html` 등)을 삭제합니다. (`.git`, `CNAME` 파일은 유지)
    *   `npx create-next-app@latest --typescript` 명령어로 Next.js, TypeScript 기반의 새 프로젝트를 생성합니다.
2.  **핵심 의존성 설치:**
    *   스타일링을 위한 `tailwindcss`를 설치하고 설정합니다.
    *   UI 애니메이션을 위한 `framer-motion`을 설치합니다.
    *   클래스 이름 관리를 위한 `clsx`와 `tailwind-merge`를 설치합니다.
3.  **디자인 시스템 적용:**
    *   `tailwind.config.ts` 파일에 기획서의 색상 팔레트, 폰트, 간격 시스템(`spacing`)을 `theme` 객체에 맞게 설정합니다.
    *   전역 스타일(`globals.css`)에 기본 배경색과 폰트 설정을 적용합니다.
4.  **기본 폴더 구조 생성:**
    *   `src/components`, `src/app`, `src/styles`, `content/posts`, `public/images` 등 기획서에 명시된 폴더 구조를 생성합니다.
5.  **메인 레이아웃 구현:**
    *   모든 페이지에서 공통으로 사용될 `src/components/layout/Header.tsx`와 `Footer.tsx` 컴포넌트의 기본 형태를 만듭니다.
    *   `src/app/layout.tsx` 파일에서 헤더와 푸터를 포함한 전역 레이아웃을 설정합니다.

**✅ 1단계 완료 체크리스트:**
- [ ] `npm run dev` 명령어로 Next.js 개발 서버가 오류 없이 실행되는가?
- [ ] 기존 `.html` 파일들이 모두 삭제되었는가?
- [ ] `tailwind.config.ts`에 커스텀 색상, 폰트, 간격이 정의되어 있는가?
- [ ] 웹사이트에 접속했을 때, 기본 배경색과 폰트가 적용된 페이지에 헤더와 푸터의 기본 형태가 보이는가?
- [ ] 기획서에 명시된 `src`, `content`, `public` 폴더 구조가 생성되었는가?

---

### **[2단계] 핵심 컴포넌트 개발 및 콘텐츠 표시**

**목표:** 메인 페이지를 구성하는 핵심 UI 컴포넌트들을 제작하고, 마크다운으로 작성된 샘플 콘텐츠를 페이지에 실제로 표시합니다.

**상세 작업 내용:**
1.  **메인 페이지 컴포넌트 제작:**
    *   `Hero.tsx`: 메인 비주얼을 보여주는 히어로 섹션 컴포넌트를 제작합니다. (반응형 이미지 포함)
    *   `TagNavigation.tsx`: 태그 필터링 네비게이션 바의 UI를 구현합니다. (기능 제외)
    *   `ContentCard.tsx`: 게시물 하나를 나타내는 카드 컴포넌트의 UI를 구현합니다.
2.  **샘플 콘텐츠 준비:**
    *   `content/posts/` 폴더 안에 `sample-post-1`, `sample-post-2`와 같은 폴더를 만듭니다.
    *   각 폴더 안에 기획서의 Front Matter 구조를 따른 `index.md` 파일을 작성하고, `featured.jpg` 이미지를 추가합니다.
3.  **콘텐츠 파싱 유틸리티 제작:**
    *   `src/utils/posts.ts` 파일을 만들어, `content/posts` 디렉토리에서 마크다운 파일들을 읽고 Front Matter 데이터를 파싱하는 함수를 구현합니다.
4.  **메인 페이지에 콘텐츠 렌더링:**
    *   `src/app/page.tsx`에서 위에서 만든 유틸리티 함수를 사용하여 모든 게시물 목록을 가져옵니다.
    *   가져온 데이터를 `ContentCard` 컴포넌트에 전달하여 메인 페이지에 게시물 그리드를 표시합니다.

**✅ 2단계 완료 체크리스트:**
- [ ] 메인 페이지에 히어로 섹션 이미지가 정상적으로 표시되는가?
- [ ] 태그 네비게이션 바가 UI적으로 구현되어 있는가?
- [ ] `content/posts`에 작성된 샘플 마크다운 게시물들이 파싱되어 메인 페이지에 카드 형태로 나타나는가?
- [ ] 콘텐츠 그리드가 화면 크기에 따라 2열, 3열 등으로 반응형으로 변경되는가?

---

### **[3단계] 상세 페이지 구현 및 동적 라우팅**

**목표:** 사용자가 메인 페이지에서 게시물 카드를 클릭했을 때, 해당 게시물의 상세 내용 페이지로 이동하고, 마크다운 콘텐츠를 올바르게 렌더링합니다.

**상세 작업 내용:**
1.  **동적 라우팅 설정:**
    *   `src/app/posts/[slug]/page.tsx` 경로에 동적 세그먼트 페이지를 생성하여 개별 포스트를 렌더링할 준비를 합니다.
2.  **상세 페이지 레이아웃 구현:**
    *   `ArticleHeader.tsx`: 상세 페이지 상단의 제목, 뒤로가기 버튼 등을 포함한 헤더를 구현합니다.
    *   `ArticleBody.tsx`: 마크다운 콘텐츠가 렌더링될 영역을 확보합니다.
    *   `ArticleMeta.tsx`: 게시 날짜, 읽는 시간, 태그 목록을 표시하는 메타 정보 컴포넌트를 구현합니다.
3.  **마크다운 렌더링:**
    *   `remark`, `remark-html`과 같은 라이브러리를 사용하여 `index.md` 파일의 본문 내용을 HTML로 변환하고, 화면에 렌더링합니다.
4.  **SEO 컴포넌트 구현 및 적용:**
    *   `src/components/SEO.tsx` 컴포넌트를 제작합니다.
    *   상세 페이지에서 게시물의 Front Matter(제목, 설명, 대표 이미지 등)를 기반으로 `<head>` 안에 동적인 메타 태그(OG 태그, Twitter 카드 등)를 생성하도록 적용합니다.
5.  **이전/다음 글 네비게이션:**
    *   게시물 상세 페이지 하단에 이전 글과 다음 글로 이동할 수 있는 네비게이션 UI를 구현합니다.

**✅ 3단계 완료 체크리스트:**
- [ ] 메인 페이지의 게시물 카드를 클릭하면 `posts/sample-post-1`과 같은 URL로 성공적으로 이동하는가?
- [ ] 상세 페이지에서 마크다운으로 작성된 본문(문단, 소제목, 이미지 등)이 HTML로 올바르게 렌더링되는가?
- [ ] 브라우저의 개발자 도구에서 `<head>` 태그를 확인했을 때, 해당 게시물에 맞는 `title`, `description`, `og:image` 등의 메타 태그가 동적으로 생성되었는가?
- [ ] 이전/다음 글 네비게이션이 표시되고, 클릭 시 다른 게시물로 이동하는가?

---

### **[4단계] 핵심 상호작용 기능 구현**

**목표:** 태그 필터링, 실시간 검색, 소셜 공유 등 사용자의 참여를 유도하는 동적인 기능들을 완성합니다.

**상세 작업 내용:**
1.  **태그 필터링 기능 구현:**
    *   `TagNavigation.tsx`에서 태그 버튼 클릭 시 `useState`를 통해 활성화된 태그 상태를 관리합니다.
    *   선택된 태그에 따라 메인 페이지의 게시물 목록을 필터링하여 보여줍니다.
    *   `framer-motion`의 `AnimatePresence`와 `layout` 속성을 사용하여 게시물 그리드가 필터링될 때 부드러운 애니메이션 효과를 적용합니다.
2.  **검색 기능 구현:**
    *   검색 버튼 클릭 시 화면 전체를 덮는 검색 오버레이 UI를 구현합니다.
    *   빌드 시점에 모든 게시물 콘텐츠로 검색 가능한 인덱스를 생성합니다.
    *   사용자가 입력창에 텍스트를 입력할 때마다 `debounce`를 적용하여 실시간으로 검색 결과를 표시하는 로직을 구현합니다.
3.  **소셜 공유 기능 구현:**
    *   `src/utils/share.ts`에 카카오톡, 트위터, 링크 복사 등 공유 로직을 함수로 작성합니다.
    *   게시물 상세 페이지에 공유 버튼들을 추가하고, 각 버튼 클릭 시 해당 함수가 실행되도록 연결합니다.

**✅ 4단계 완료 체크리스트:**
- [ ] 메인 페이지에서 특정 태그를 클릭하면 해당 태그를 가진 게시물만 애니메이션과 함께 표시되는가?
- [ ] 검색 오버레이가 열리고, 검색어를 입력하면 관련 게시물 목록이 나타나는가?
- [ ] 게시물 상세 페이지에서 '링크 복사' 버튼을 누르면 클립보드에 현재 URL이 복사되는가?
- [ ] (선택) 카카오톡 공유 API가 연결되어 있다면, 공유 버튼이 정상적으로 작동하는가?

---

### **[5단계] 최적화, 배포 자동화 및 최종 검수**

**목표:** 웹사이트의 성능을 극대화하고, GitHub Actions를 통해 배포를 자동화하며, 최종 품질을 검수합니다.

**상세 작업 내용:**
1.  **성능 최적화:**
    *   `useInfiniteScroll` 커스텀 훅을 구현하여 메인 페이지에 무한 스크롤 기능을 적용합니다.
    *   `next/image` 컴포넌트를 적극적으로 사용하여 이미지를 WebP 포맷으로 자동 변환하고, 사이즈 최적화를 수행합니다.
2.  **Sitemap 생성:**
    *   빌드 시점에 `content/posts`를 기반으로 `sitemap.xml`을 동적으로 생성하는 스크립트(`scripts/generate-sitemap.js`)를 작성하고, `package.json`의 `build` 명령어에 포함시킵니다.
3.  **배포 자동화 (CI/CD):**
    *   `.github/workflows/deploy.yml` 파일을 작성합니다.
    *   `main` 브랜치에 코드가 푸시될 때마다 자동으로 Next.js 프로젝트를 빌드하고, 결과물을 `gh-pages` 브랜치에 배포하도록 설정합니다.
4.  **최종 검수:**
    *   Chrome Lighthouse를 사용하여 성능, 접근성, SEO 점수를 측정하고, 개선이 필요한 부분을 수정합니다.
    *   다양한 디바이스(모바일, 태블릿, 데스크톱)에서 레이아웃이 깨지지 않는지 최종 확인합니다.
    *   프로젝트 실행 및 콘텐츠 관리 방법을 설명하는 `README.md` 파일을 상세하게 작성합니다.

**✅ 5단계 완료 체크리스트:**
- [ ] 메인 페이지에서 아래로 스크롤하면 다음 페이지의 게시물들이 자동으로 로딩되는가?
- [ ] 프로젝트 빌드 후 `public` 폴더 안에 `sitemap.xml` 파일이 생성되었는가?
- [ ] `main` 브랜치에 푸시했을 때 GitHub Actions가 자동으로 실행되어 `jihotv.site`에 변경사항이 반영되는가?
- [ ] Lighthouse 측정 결과, 모든 항목에서 90점 이상의 높은 점수를 획득했는가?
- [ ] 최종 `README.md` 파일이 잘 작성되었는가?
